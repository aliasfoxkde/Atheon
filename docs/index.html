atheon

Secret detection for Java. A library you embed. A CLI you run anywhere.

Atheon scans for leaked API keys, tokens, and credentials in code, config files, and environment variables. One dependency. No subprocess. No native binary. Runs wherever a JRE exists.

$ java -jar atheon-1.0.1-cli.jar scan .

[CRITICAL] aws-access-key
  file:  src/config/dev.properties:14
  match: AKIA**********************MPLE

─────────────────────────────
found 1 potential secret(s)

files: 42  size: 318.7 KB  time: 84ms

install

Add the JitPack repository and declare the dependency:

<repositories>
  <repository>
    <id>jitpack.io</id>
    <url>https://jitpack.io</url>
  </repository>
</repositories>

<dependency>
  <groupId>com.github.HoraDomu</groupId>
  <artifactId>Atheon</artifactId>
  <version>v1.0.1</version>
</dependency>

Or download the CLI JAR from GitHub Releases and run it anywhere a JRE is installed.

use it

Runner runner = new Runner(new Registry());

runner.scanString("AKIAIOSFODNN7EXAMPLE");
runner.scanFile(Path.of("config.yaml"));
runner.scanDir(Path.of("/path/to/project"));
runner.scanEnv();

Each Finding exposes scanner, severity, file, line, and match. That is the entire API.

cli

# scan a directory
java -jar atheon-1.0.1-cli.jar scan .

# scan env vars
java -jar atheon-1.0.1-cli.jar scan --env

# pipe git diff
git diff | java -jar atheon-1.0.1-cli.jar scan --stdin

# json output
java -jar atheon-1.0.1-cli.jar scan . --json

Exit code 0 is clean. Exit code 1 means findings. Runs on any OS, any architecture, any CI — if a JRE is there, Atheon works.

vs gitleaks

gitleaks is excellent. Atheon does not compete with it.

Use gitleaks to audit git history. Use Atheon when you need secret detection inside a Java application — scanning a string just read from a config, checking environment variables before they reach your code, or embedding checks into a build step without shelling out to a native binary.

mission

Secret detection should be native to Java development — not a binary you wrap in a shell script, not a CI step that requires a separate install. A library you call like any other, embedded in your code, running wherever Java runs.

Atheon is feature complete. The API, CLI, output formats, and exit codes are stable and will not change. Future releases add scanners and security patches only.
