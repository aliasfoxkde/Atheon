leakr

Open-source secret detection for Java.

Leakr detects leaked API keys, tokens, and credentials in your code, config files, and environment variables. Add one dependency, call one method. No subprocess, no binary, no configuration.

  • Embeddable Java library — call it directly from your application code
  • Scans strings, files, directories, and live environment variables
  • JRE only — one JAR, any platform, no native binary
  • Parallel directory scanning sized to the host CPU automatically

Install

<dependency>
  <groupId>io.github.horadomu</groupId>
  <artifactId>leakr</artifactId>
  <version>1.0.0</version>
</dependency>

Use it

Runner runner = new Runner(new Registry());

runner.scanString("AKIAIOSFODNN7EXAMPLE");
runner.scanFile(Path.of("config.yaml"));
runner.scanDir(Path.of("/path/to/project"));
runner.scanEnv();

Each Finding exposes scanner, severity, file, line, and match. That's the entire API.

Leakr and gitleaks

gitleaks is a great tool and Leakr isn't trying to compete with it. They solve different problems.

If you want to scan your git commit history for secrets that were accidentally checked in, use gitleaks. It was built for that and does it well.

Leakr is for when you need secret detection inside a Java application — not as a separate tool you shell out to, but as a library you call directly. Scan a string your app just read from a config file. Scan environment variables at service startup before they reach your code. Scan a directory as part of a build step with no binary to install and no architecture to worry about. That is the gap Leakr fills.

CLI

Leakr also ships as a self-contained CLI JAR for scanning without writing any code. Download from GitHub Releases or build from source:

git clone https://github.com/HoraDomu/Leakr.git
cd Leakr && mvn package -q
java -jar target/leakr-1.0.0-cli.jar scan .

Exit code 0 means clean. Exit code 1 means findings. Use --json for machine-readable output. Pipe git diff through it with --stdin.

Mission

Secret detection should be a native part of Java development — not a binary you wrap in a shell script, not a CI step that requires a separate tool install. A library you call like any other, embedded in your code, running wherever Java runs, requiring nothing beyond a JRE.

Leakr is feature complete. The CLI, library API, output formats, and exit codes are stable and will not change. Future releases add scanners and security patches only.
