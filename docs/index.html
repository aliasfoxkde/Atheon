leakr

open-source secret scanner for Java

Leakr is an open-source secret and credential scanner for Java. Maintained by Dominick Yanez aka HoraDomu,It's one JAR, any platform. Scan files, directories, environment variables, and stdin for leaked API keys, tokens, and credentials, or embed it directly in your Java application with no subprocess and no native binary required.

built-in scanners, with the goal to adding more. Since this is a feature complete product the scanners add width rather than depth, and allow the Scans to be more complete and detect keys and secrets that could lead to catastrophic consequences. If there are any I forgot to add please contact me about them, I will try to cover the most important ones. All scanners are fully tested..

Install

curl -L https://github.com/HoraDomu/Leakr/releases/download/v1.0.0/leakr.jar -o leakr.jar
java -jar leakr.jar scan .

Exit code 0 means clean. Exit code 1 means findings. Pipe-friendly with --json output for CI. Pipe git diff straight through it with --stdin. Build from source by cloning the repo and running mvn package -q.

Mission

Secret detection should be a native part of Java development. Not a binary you wrap in a shell script, not a CI step that requires a separate tool. A library you call like any other, embedded in your code, running wherever Java runs, requiring nothing beyond a JRE.

Leakr is feature complete. New features are not being merged. Future releases will be security patches and new scanners only.
