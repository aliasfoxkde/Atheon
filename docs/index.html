Atheon

Secret detection, natively Java. One dependency. No subprocess. No binary. Runs wherever a JRE exists.

Atheon detects leaked API keys, tokens, and credentials in your code, config files, and environment variables. Add one dependency, call one method. No config files, no plugin system, no architecture-specific binary to bundle.

Embedded library

Call it directly from Java. Scan strings, files, directories, or live environment variables — all in-process.

CLI tool

Self-contained JAR. java -jar atheon.jar scan . — no install, no PATH entry, no native binary.

Zero config

No YAML. No rule files. Drop it in, call it, get results. Scanners auto-register at runtime.

Parallel scanning

Directory scans use a thread pool sized to the host CPU. Fast on any hardware.

Install

<dependency>
  <groupId>io.github.horadomu</groupId>
  <artifactId>atheon</artifactId>
  <version>1.0.1</version>
</dependency>

Use it

Runner runner = new Runner(new Registry());

runner.scanString("AKIAIOSFODNN7EXAMPLE");
runner.scanFile(Path.of("config.yaml"));
runner.scanDir(Path.of("/path/to/project"));
runner.scanEnv();

Each Finding exposes scanner, severity, file, line, and match. That's the entire API surface.

Runs everywhere

A JRE is the only requirement. No native binary, no architecture matrix, no install step. Ship Atheon in a Docker container, run it in a GitHub Action, embed it in a Spring Boot service, call it from a Gradle task — it does not care.

Windows macOS Linux ARM64 Docker GitHub Actions GitLab CI Jenkins

Atheon vs. gitleaks

gitleaks is a great tool and Atheon isn't trying to replace it. They solve different problems.

Use gitleaks to audit your git history — it was built for that. Use Atheon when you need secret detection inside a Java application: scan a string your service just read from an external config, validate environment variables at startup before they touch your code, check a user-uploaded file before it gets stored. That is the gap Atheon fills, and no CLI tool can fill it.

CLI

Atheon also ships as a self-contained CLI JAR. Download from GitHub Releases or build from source:

git clone https://github.com/HoraDomu/Atheon.git
cd Atheon && mvn package -q
java -jar target/atheon-1.0.1-cli.jar scan .

Exit code 0 is clean. Exit code 1 means findings. Use --json for machine-readable output. Pipe git diff through it with --stdin.
